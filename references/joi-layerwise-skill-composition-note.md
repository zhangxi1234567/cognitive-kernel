# JOI Teaching Sample -> General Layerwise Skill Composition

This note records the teaching signal extracted from the JOI-style walkthrough.

JOI is not the target here.

JOI was the teaching sample.

The real target is a general solve rhythm that should transfer across problems and domains.

## Scope

This is not a JOI-only note.

This note exists because the JOI walkthrough was used to teach the runtime how to solve all later problems.

So:

- JOI is the demonstration case
- layerwise skill composition is the transferable method
- later problems should use the same method, not merely remember the JOI surface route

The intended transfer target includes:

- programming problems
- grammar / construction problems
- math problems
- physics-style problems
- research-style reasoning
- any other problem where the surface shell is thicker than the true controller

## Core Rule

Do not use one skill combination once at the beginning and then fall back to ordinary solving.

Do not treat "compression succeeded" as permission to stop choosing primitives.

Even after the object has been compressed, the reduced object must still be solved by further skill composition.

In other words:

1. peel one layer
2. re-evaluate the live object
3. choose the next primitive combination again
4. pressure-test the conversion
5. continue until final settlement is closed

## What Was Actually Taught

The JOI lesson was:

- not "pick one global trick for the whole problem"
- but "at each new layer, re-select the best primitive combination"

The model must repeatedly ask:

- Is the current object still a fake surface?
- Is there a smaller truthful carrier now?
- Is the next pressure symmetry, conservation, projection, assignment, contradiction, state conversion, or interface closure?
- Did this conversion preserve the original task?
- Did a local truth get mistaken for a global truth?
- Is the current carrier already target-complete, or is it only a pretty middle object?

## Layerwise Primitive Pattern

Typical sequence taught by the JOI walkthrough:

1. surface suspicion
   - doubt the presented object
   - refuse inertia on the literal shell

2. carrier replacement
   - convert the thick object into a thinner truthful carrier
   - prefer relation, readout, projection, or compressed state over raw story

3. representation replacement
   - once one shell is removed, choose again
   - do not keep pushing the first representation by habit

4. global settlement pressure
   - do not stop at a local checker
   - compress into a globally countable or globally sealable object

5. adversarial back-questioning
   - after each conversion, ask whether the converted problem is still the same problem
   - ask whether boundaries, multiplicity, overlap, or compatibility were lost

6. final interface closure
   - the last layer is not "ordinary finishing"
   - the last layer still requires skill composition
   - settlement itself is another layer of primitive choice

## Ordered JOI Teaching Sequence

What was taught was not a bag of JOI tricks.

It was an ordered solve rhythm.

The order matters.

### Step 1: Draw First When The Object Is Still Foggy

- externalize the object
- turn verbal fog into visible structure
- use drawing to see adjacency, separation, crossing, corridor, blocking, and ownership

This was the first move when the object was still too thick to trust verbally.

### Step 2: Use Contradiction To Kill Fake Structure

- after drawing, ask whether a proposed interpretation creates an impossible structure
- test whether the object would become something the original family cannot legally be
- use structural impossibility early instead of pushing a fake route

This was not late verification.

It was an early route-killer.

### Step 3: Use Symmetry And Balance To Remove Fake Freedom

- once the visible structure exists, ask what is really swappable
- ask whether balance reveals the controlling location
- use symmetry and equilibrium to delete fake directional bias

This is where "symmetry" and "balance" were taught as freedom-killers, not slogans.

### Step 4: Use Limit / Degeneration / Edge Push To Expose Skeleton

- push the object to coincidence, touching, zero, saturation, collapse, or edge behavior
- if the thick form hides the mechanism, squeeze it until the mechanism is forced out
- treat boundary and degeneration as ways to reveal the honest carrier

This came after the first structural cleaning, not before.

### Step 5: Use Singularity / First-Crack To Find The Weakest Seam

- ask where a wrong route breaks first
- find the earliest crack, unstable seam, branch switch, or singular point
- use the first failure point as the cheapest honesty test

This was the "explosion point" layer.

### Step 6: Use Assignment / Special Value As A Controlled Probe

- choose a lawful value or special case
- do not plug randomly
- use assignment only to expose mechanism, compress candidates, or kill a route

This came after enough structure existed to know what was worth probing.

### Step 7: Use Dimension Shift / Re-encoding When The Current Carrier Is Still Thick

- if the object is still bulky, change carrier again
- flatten, project, lift, convert, or state-encode
- do not remain in an old shell just because it was useful one layer ago

This is where you kept forcing me to re-select the next representation instead of staying on inertia.

### Step 8: Re-Ask The Primitive Choice At The New Layer

- after every successful peel, stop and choose again
- do not stretch the previous primitive farther than it deserves
- ask what should wake now on this thinner carrier

This is the most important rule:

the same problem must re-select primitives layer by layer.

### Step 9: Final Settlement Still Uses Skill Composition

- after compression, do not switch back to ordinary default solving
- the reduced object still needs active primitive selection
- final closure is also a primitive layer

This was taught explicitly:

the endgame is not outside the skill system.

## Actual Ordered Primitive Chain

In the JOI teaching sequence, the live order was approximately:

1. draw
2. contradiction
3. symmetry / balance
4. limit / degeneration
5. singularity / first-crack
6. assignment / special value
7. dimension shift / re-encoding
8. re-select primitives again on the new layer
9. use another primitive combination for final settlement

## Generalization Rule

The above order came from the JOI teaching sample,
but it must not stay attached only to JOI.

The intended general rule is:

- when a new problem appears, do not ask "what was the JOI answer?"
- ask "what layer am I on now, and which primitive should wake now?"

So the transferable part is:

- ordered, layer-sensitive primitive selection
- repeated re-selection after each peel
- no return to ordinary default solving after early compression
- final settlement still lives inside the same primitive system

## Non-Negotiable Reminder

The important thing is not that every problem must use the exact same visible order.

The important thing is:

- the order is layer-sensitive
- the primitive must be re-chosen after each peel
- the final compressed object must still be solved by further skill composition

So the real lesson extracted from the JOI teaching sample was:

ordered, recursive, layerwise skill composition,
all the way to closure.

## Non-Negotiable Rule

One problem may require many rounds of skill composition.

It is a regression if the model:

- uses a skill only once
- compresses to a new object
- then solves the rest in ordinary default mode without re-selecting primitives

The intended behavior is:

- every layer can wake a different combination
- the combination may change multiple times inside one problem
- the solve is only complete when the final interface is honestly sealed

## Practical Reminder

When a candidate route looks promising, do not ask only:

- "Did I find a good route?"

Also ask:

- "What is the next layer now?"
- "What new primitive should wake on this thinner carrier?"
- "Is this already the final owner, or only the next shell?"

## Short Version

The JOI teaching signal was:

not one trick per problem,
but repeated primitive re-selection per layer,
all the way down to final settlement.
