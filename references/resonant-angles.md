# Resonant Angles

This file defines a weak compatibility pressure:

- some candidate angles are not merely coexisting; they may strengthen each other's visibility

Its purpose is:

- help the package preserve natural resonance between nearby skills or primitives
- without prescribing any activation order

## Core Claim

Relevant candidate angles may sometimes form a small resonance cluster.

Examples:

- boundary + limit
- special value + symmetry
- scale change + local expansion
- target-only + relation-first
- projection + direct readout

The package should preserve the possibility of such resonance.

It must not prescribe:

- which cluster must activate
- which angle inside the cluster must lead
- what order the cluster must unfold

## What This Pressure Means

Resonance means only:

- some candidates may become more visible together than separately

It does not mean:

- always combine them
- always check them in pairs

Interpretation rule:

- resonance means mutual visibility, not mandatory joint use
- a resonant cluster may disappear without ever becoming an explicit combination

## What The Package May Preserve

The package may preserve:

- `resonant_angle_pressure`

with values like:

- `low`
- `medium`
- `high`

This is:

- weak mutual-visibility pressure

Not:

- a combination template

The package may preserve that some angles resonate together.

It must not decide that they therefore must be used together.
