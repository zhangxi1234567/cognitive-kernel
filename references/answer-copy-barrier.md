# Wu Boshi Answer-Copy Barrier

This file defines the test:

- if the user were given the answer first, could they reconstruct the route clearly?

Main idea:

- many standard solutions produce an answer
- but do not produce a route that a learner can genuinely copy, reuse, or rebuild

Wu Boshi style should break that barrier.

## Core Principle

A good reduction route should not only end at the right answer.
It should make the answer *copyable*:

- the learner sees why this answer shape appears
- the learner sees what move forced it
- the learner could replay the route on a nearby problem

## Failure Pattern

An answer-copy barrier is present when:

- the final answer is clean
- but the route still feels like hidden magic
- if the learner were handed the answer, they still would not know how to justify it

## Success Pattern

The barrier is broken when:

- the answer shape is tied to a visible primitive
- the key hinge is short enough to replay
- the legality mechanism is local and nameable
- the learner could copy not only the result, but the route pattern

## Trigger Questions

Ask:

1. If I showed the user the final answer first, could they now see the route?
2. What exact primitive makes this answer plausible?
3. What short hinge would let them rebuild the path?
4. Is the route answer-shaped, or still chapter-shaped?

## Where It Transfers

- school problem solving
- olympiad and exam training
- university proof patterns
- research toy-model conjecturing
- debugging from expected behavior backward

## Hard Rule

Do not be satisfied that the answer is elegant.
Ask whether the route has become copyable.
