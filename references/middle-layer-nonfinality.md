# Middle-Layer Nonfinality

This file defines a weak compatibility pressure:

- elegant middle-layer success should not feel fully final while a deeper useful controller still plausibly exists

Its purpose is:

- keep the package from mistaking one good rewrite for the deepest useful carrier
- preserve the attraction of a smaller truthful object
- do so without turning depth into a scripted descent ladder

## Core Claim

Some routes become:

- cleaner
- smarter
- more elegant

without yet becoming:

- deepest useful
- smallest honest
- truly load-bearing enough to settle

The package should preserve that distinction.

## What This Pressure Means

Middle-layer nonfinality means only:

- the current route may be good but not yet deep enough

It does not mean:

- now peel one more layer
- now ask another why-question
- now run another skill round

Those decisions remain model-side.

## Typical Signals

This pressure is relevant when:

- one carrier improved but still feels composite
- the route became elegant faster than it became small
- the target is still being paid through a heavier shell than necessary
- nearby lower primitives still seem plausibly reachable

## What The Package May Preserve

The package may preserve:

- `middle_layer_nonfinality`

with values like:

- `low`
- `medium`
- `high`

This is:

- weak non-terminality pressure

Not:

- a forced continuation command

The package may preserve suspicion that middle-layer success is not yet final.

It must not decide how that suspicion is resolved.
