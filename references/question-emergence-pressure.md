# Question Emergence Pressure

This file defines a weak compatibility pressure:

- the current route may deserve renewed scrutiny without the package specifying any explicit question script

Its purpose is:

- move the package closer to self-arising internal questioning
- avoid turning questioning into an external checklist

## Core Claim

The package cannot safely tell the model:

- what exact question to ask next

But it can preserve a weaker thing:

- whether the current route deserves renewed scrutiny

That is enough to support more natural internal questioning,
without proceduralizing thought.

## What This Pressure Means

Question-emergence pressure means:

- the current route may still be too thick
- the current carrier may still be composite
- a deeper controller may still plausibly exist
- nearby weak candidates may still matter

It does not mean:

- now ask a why-question
- now recurse
- now peel another layer

## What The Package May Preserve

The package may preserve a weak signal such as:

- `question_emergence_pressure`

with values like:

- `low`
- `medium`
- `high`

This is:

- a trigger for internal re-examination

Not:

- a trigger for external script execution

## Relation To Existing Pressures

This pressure can be informed by:

- `remaining_fake_burden`
- `premature_settling_risk`
- `composite_shell_risk`
- `bottom_layer_priority`
- `low_floor_access_priority`

It does not replace them.

It summarizes when those pressures together are strong enough that the route should not feel fully settled yet.

## What Must Not Happen

Do not turn this pressure into:

- fixed self-question prompts
- mandatory recursive loops
- visible “go deeper now” commands

## Final Rule

The package may preserve the pressure that a better question may still be needed.

It must not decide what that question is.
