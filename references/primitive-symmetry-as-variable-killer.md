# Primitive: Symmetry As Variable Killer

## Pattern

A problem looks multi-case or multi-variable, but symmetry makes several distinctions fake.
Once the symmetry is spent, the search space collapses.

## Trigger Signal

- left/right, above/below, before/after, or label-swapped roles are structurally interchangeable
- the clean answer shape suggests balanced placement, midpoint structure, or equal-offset framing
- the standard route duplicates the same work on mirrored or relabeled branches
- extrema, range, or construction questions seem likely to lock at symmetric positions
- the picture, algebra, or function law visibly preserves a reflection, center, relabeling, or balanced frame

## Level-Agnostic Core

Symmetry is not decoration.
It is a variable killer and case-count reducer.

## Control Mechanism

The move works because:

- symmetric objects share the same control law
- relabeling or reflection preserves the target while deleting redundant distinctions
- balanced placement often exposes one midpoint, axis, or center that controls several moving parts at once
- symmetric extremal positions frequently collapse free variables into one representative configuration

## Preserved Structure

Typical preserved items:

- equality of mirrored roles
- fixed midpoint, axis, or center
- unchanged target under relabeling / reflection
- reduced case count without losing admissible configurations

## Earliest Honest Layer

Before computing, ask:

- which differences here are fake because of symmetry?
- what single representative position can stand in for several cases?
- where is the axis, midpoint, center, or balanced frame that kills work?

## Fast Move

1. Name the exact symmetry: mirror, center, relabel, balanced frame, or periodic symmetry.
2. State what work disappears because of it.
3. Reduce the problem to one representative case, one axis, or one balanced position.
4. Solve the reduced problem and only add a seam check if symmetry does not settle everything.

## Compression Move

Typical burden deletion:

- many cases -> one representative case
- many variables -> one balanced coordinate or offset
- asymmetric-looking shell -> symmetric control frame
- duplicated work -> one shared calculation

## Level Transfer

- low-complexity form: mirror picture, midpoint, equal sides, equal offsets
- compact formal form: symmetry axis, center symmetry, relabel invariance, odd/even or periodic structure
- advanced formal form: invariant representative, gauge choice, quotient by symmetry, balanced extremal frame

## Cross-Domain Analogue

- debugging: merge equivalent failure branches and inspect one representative path
- product/strategy: collapse equivalent user segments or mirrored scenarios before over-modeling
- science: use symmetry reduction to eliminate fake degrees of freedom before full analysis

## Where It Transfers

- geometry extrema and constructions
- function parity, periodicity, and graph behavior
- combinatorics through relabel symmetry
- vector and coordinate problems with balanced placements
- optimization and range problems where symmetric frames expose the active mechanism

## Failure Mode

- the symmetry is only cosmetic and does not preserve the target
- an asymmetric constraint quietly breaks the symmetry
- symmetric placement suggests the answer shape but does not uniquely force it
- using symmetry deletes a branch that actually matters

## Verification Bill

State:

- what exact symmetry is present
- what work is legally deleted
- what still needs checking because symmetry alone does not decide it
