# Wu Boshi Isomorphic Shape Families

This file defines the move:

- recognize that apparently different function forms belong to the same behavior family after a small transformation

## Core Principle

Some function playlists are really teaching:

- one behavior shape
- many surface disguises

These disguises may come from:

- translation
- scaling
- sign change
- multiplication / division
- add-subtract patching
- local piece replacement

## Common Public Labels

- `顺反同构`
- `平移拉伸后的同构函数`
- `乘除导致的凹凸反转同构函数`
- `加减同构`
- `局部同构`
- `差一同构`

## Fast Move

1. Ignore surface clutter.
2. Ask what transformation connects this object to a mother form.
3. Route the problem by inherited behavior, not local ugliness.

## Hard Rule

Do not call two objects isomorphic if the transformation changes the target-relevant behavior.
