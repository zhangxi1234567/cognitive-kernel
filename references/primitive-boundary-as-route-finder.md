# Primitive: Boundary As Route Finder

## Pattern

The edge case is not only a final check.
It is the cleanest place to discover the active mechanism, answer shape, or controlling relation.

## Trigger Signal

- the target asks for range, threshold, extremum, onset, feasibility, or parameter interval
- the full interior looks messy but endpoint, tangent, vertex, equality, or infinite-limit cases look structurally clean
- several candidate routes compete, but one boundary case seems able to separate them quickly
- the problem feels global while one first dangerous edge already looks load-bearing
- the answer shape seems likely to be locked by a boundary configuration

## Level-Agnostic Core

Do not treat the boundary as a late checksum only.
Often the boundary is the cheapest route-discovery device.

## Control Mechanism

The move works because:

- the edge case preserves the decisive control relation more clearly than the interior
- onset, saturation, tangency, equality, or extreme-position configurations often expose the governing seam
- many interior behaviors are organized by what happens at the first boundary where the mechanism locks or breaks

## Preserved Structure

Typical preserved items:

- threshold order
- regime split
- target-relevant ratio or alignment at the edge
- seam logic connecting the edge to the interior
- limiting relation that still controls the original target

## Earliest Honest Layer

Before solving the whole interior, ask:

- where does this structure become simplest without becoming irrelevant?
- which edge or limit would reveal the mechanism first?
- what answer shape or route becomes visible there?

## Fast Move

1. Identify the most informative boundary: endpoint, tangent, equality case, limiting case, or extremal position.
2. Read what mechanism becomes visible there.
3. Use that edge to guess the route, threshold, or answer shape.
4. Map back to the interior only as much as the verification bill requires.

## Compression Move

Typical burden deletion:

- full interior analysis -> edge-mechanism discovery
- many candidate routes -> one boundary-selected route
- broad parameter search -> seam or endpoint check
- global shape analysis -> onset / tangency / extreme-position readout

## Level Transfer

- low-complexity form: check endpoints, equal cases, or obvious extreme positions first
- compact formal form: tangent case, equality case, saturation point, boundary value
- advanced formal form: asymptotic regime edge, instability onset, locking boundary, limiting configuration

## Cross-Domain Analogue

- debugging: inspect the first failing boundary condition before modeling every normal-state behavior
- strategy: learn the system by the constraint edge where it breaks or saturates
- science: identify the regime boundary before solving every interior detail

## Where It Transfers

- parameter ranges
- conic and geometry extrema
- inequalities and sign conditions
- optimization and feasibility questions
- function thresholds and regime switches
- any problem where the edge reveals the route sooner than the interior

## Failure Mode

- the chosen boundary is easy but not controlling
- the limiting configuration destroys the target-relevant structure
- several competing boundaries exist and only one was checked
- the edge suggests the answer shape but does not force it

## Verification Bill

State:

- why this boundary is informative rather than accidental
- what is preserved at the edge
- what part of the interior still needs a minimal seal

## Seed Examples

- conic eccentricity ranges read from `e -> 0`, `e -> 1`, tangent, or vertex configurations
- parameter problems where the minimum/maximum occurs at a just-saturated condition
- geometry questions where tangent or endpoint placement reveals the controlling ratio
- function and inequality problems where equality or boundary cases reveal the route
