# Composite Shell Pressure

This file defines a weak compatibility pressure:

- the current route may still be operating on a composite shell rather than the deepest useful controller

Its purpose is:

- keep the package from rewarding beautifully compressed middle layers too early
- do so without teaching a decomposition script

## Core Claim

Some routes look elegant because they already compressed one layer.

But they may still be:

- composite
- layered
- not yet at the most fundamental useful carrier

The package should preserve that possibility as pressure.

It must not prescribe how to resolve it.

## What This Pressure Means

This pressure means only:

- the current carrier may still hide a deeper shared controller

It does not mean:

- now peel another layer
- now ask a new why-question
- now run a recursive loop

## What The Package May Preserve

The package may preserve:

- `composite_shell_risk`
- `bottom_layer_priority`
- `carrier_integrity`
- `coexisting_weak_candidates`

These pressures can coexist.

They must remain:

- weak
- descriptive
- non-binding

## What The Package Must Not Do

It must not define:

- shell taxonomies the model must replay
- layer-counting rituals
- mandatory descent ladders

## Relation To Existing Docs

Use:

- `bottom-layer-priority.md`
- `carrier-integrity-check.md`
- `anti-premature-settling.md`
- `anti-overdefinition-of-essence.md`

This file adds one emphasis:

- middle-layer success may still be composite-shell success

## Final Rule

The compatibility layer may keep a weak suspicion that the current route is still composite.

It must not tell the model how that suspicion must be resolved.
