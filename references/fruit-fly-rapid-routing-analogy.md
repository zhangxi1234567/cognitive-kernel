# Fruit-Fly Rapid Routing Analogy

This note is not evidence that human mathematical solving literally works like a
fly brain.

It is a design analogy for this project's runtime:

- why a trained solver can see a problem and very quickly know
  which local combination should wake now
- without running a long explicit pipeline first

## What To Borrow

The useful high-level analogy from Drosophila work is:

- sparse compression happens early
- value / salience is attached locally, not only globally
- competing output channels are biased quickly by context and prior learning
- action selection is therefore fast because the system is not recomputing from scratch

This is the runtime lesson, not a biological identity claim.

## Project Translation

In this project, that suggests:

1. do not solve broadly and then label afterward
2. compress the object early into a sparse live carrier
3. let trained local combinations compete on that carrier
4. bias competition with remembered mechanism families
5. require one local owned bite before narrative continuation

## Why The User Can See The Next Step So Fast

The user's speed is not "memorized pipelines."

It is closer to:

- fast carrier compression
- fast singular-point / seam / symmetry salience detection
- immediate local competition among a few already-trained combinations
- near-instant inhibition of combinations that do not fit the live carrier

So the subjective feeling is:

- not "I followed many steps"
- but "I instantly knew the right local combo"

## Runtime Implication

The runtime should therefore prefer:

- salience competition over broad route narration
- local combo authorization over post-hoc naming
- re-selection at each thinner layer
- partner-supported combinations over broad single-skill claims

And it should refuse:

- ordinary solving first, labeling later
- broad primitives claiming ownership alone with weak evidence
- treating one teaching sample as a fixed workflow

## Non-Negotiable Reminder

This analogy must not become:

- a neuroscience costume
- a pipeline myth
- a justification for vague intuition without owned local bites

The point is only:

- fast compressed carrier
- fast local competition
- fast inhibition
- explicit local takeover
