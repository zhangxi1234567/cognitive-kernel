# Anti-Premature Settling

This file defines a hard rule:

- the compatibility layer must not reward stopping at a merely convenient middle layer

Its purpose is:

- keep the package biased toward deeper truthful controllers
- without externally scripting how the model must descend

## Core Claim

The package should not make the model comfortable with:

- one locally useful reduction
- one elegant middle layer
- one partly compressed route

if a smaller shared controller is still likely available.

At the same time, the package must not externally force:

- another recursive move
- another why-question
- another named-skill step

So the rule is not:

- always go one layer deeper

The rule is:

- do not let the package accidentally sanctify a still-premature stopping point

## What Premature Settling Looks Like

Examples:

- a route finds a local trick but not the shared carrier
- a route improves wording but not controller depth
- a route reaches a useful symmetry picture but not the decisive readout
- a route reaches one reduction and then silently regrows

These are not necessarily wrong routes.

They are wrong only when the package treats them as final too early.

## What The Package May Do

The package may:

- bias against still-thick carriers
- preserve memory that a route may still be premature
- preserve tension that a shared controller may still exist
- reward routes that actually delete more burden

The package may not:

- prescribe an exact deeper question
- prescribe an exact next decomposition move
- prescribe a fixed recursive ladder

## Relation To State

The package may preserve a pressure signal like:

- `remaining_fake_burden`

But that signal must remain:

- descriptive
- provisional

Not:

- a command to recurse

## Relation To Route Memory

The package may remember:

- this route looked good but settled too early
- this route found a middle layer but not the shared controller

That memory is good.

What is not good is turning that memory into:

- a future mandatory extra step

It must not tell the model exactly how to go deeper.
