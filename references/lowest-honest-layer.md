# Wu Boshi Lowest Honest Layer

This file defines one of the deepest operational doctrines of the project:

- high formal surface does not automatically mean high controlling mechanism

Many hard-looking problems are not hard because the underlying controller is advanced.
They are hard because the controller is dressed in a higher shell.

The skill should therefore keep asking:

- what is the lowest honest layer at which the controlling mechanism can still be rebuilt without cheating?

That is the layer where understanding should begin.

## 1. Not The Simplest Layer, But The Lowest Honest Layer

The target is not:

- the cutest explanation
- the most childish analogy
- the lowest possible school label

The target is:

- the earliest layer where the real controller is still visible
- the reduction is still lawful
- the decisive hinge can still be executed rather than merely admired

This is why the project should say:

- lowest honest layer

not:

- easiest layer
- elementary layer
- baby version

## 2. The Core Principle

If a high-level object and a lower-level object share the same control mechanism, then the solver should prefer to work at the lower honest layer first, and only climb back upward when needed for rigor, notation, or final sealing.

This doctrine blocks two opposite mistakes:

### Mistake A: Prestige inflation

- the surface looks advanced
- so the mechanism is wrongly assumed to be advanced

### Mistake B: Fake simplification

- the surface is rewritten in easier language
- but the real hinge is still secretly imported from the harder route

The correct move is:

- identify the shared controller
- check what is preserved
- work at the lowest layer that still carries the hinge

## 3. Canonical Examples

### Derivative

Bad reduction:

- "derivative is just velocity"

Good reduction:

- derivative and velocity share the same local-rate-of-change controller
- first understand the local change law
- then restore the formal derivative machinery when needed

### Second derivative

Bad reduction:

- "second derivative is just acceleration"

Good reduction:

- second derivative and acceleration share change-of-change structure
- the lower honest layer is local motion curvature in behavior, not merely the word "acceleration"

### Conics

Bad reduction:

- "ellipse is just a circle"

Good reduction:

- some conic problems can be normalized, completed, or routed back to a circle-like mother mechanism
- only do this when the preserved relation survives the rewrite

### Triangle extrema

Bad reduction:

- "all max/min triangle questions are elementary symmetry"

Good reduction:

- many extremum problems are controlled by balance, symmetry, equalization, or boundary structure
- the solver should test whether symmetry is the true controller before firing it

### Matrices

Bad reduction:

- "matrices are just elementary equations"

Good reduction:

- matrices and equation systems may share a linear-constraint controller
- work first at the linked-constraint layer, then recover basis, transformation, or operator language when needed

## 4. Cross-Age Transfer Rule

The same mechanism may appear as:

- a picture for a child
- a balance relation for a teenager
- a structural rewrite for an undergraduate
- a reduced model for a researcher

These are not separate methods.
They are different honest surfaces of the same controller.

Age changes:

- wording
- notation
- trust bill
- explicitness

Age does not automatically change:

- the controller
- the preserved structure
- the legality basis

## 5. Cross-Disciplinary Rule

The same controller may reappear across:

- mathematics
- physics
- chemistry
- biology
- debugging
- strategy

Examples of shared deep controllers:

- conservation
- equilibrium
- linear coupling
- local-rate-of-change
- dominant mechanism
- threshold / boundary onset
- symmetry / invariance
- aggregate control

So the first classification should not be:

- "this is math"
- "this is physics"
- "this is chemistry"

The first classification should be:

- what controller-world is this in?

## 6. Controller Worlds

Many cross-domain problems can be routed first by controller world rather than subject label.

Examples:

- conservation world
- equilibrium world
- symmetry world
- boundary-revelation world
- dominant-mechanism world
- constraint-coupling world
- target-readout world
- instability world

This is one of the keys to cross-disciplinary transfer.

## 7. Operational Test

A claimed lowest-honest-layer reduction passes only if:

1. the same controller is still active
2. the decisive hinge is still executable
3. the preserved structure is named
4. the legality basis is named
5. the route does not secretly import the old harder hinge
6. the lower layer genuinely changes variable choice, route order, or proof burden

If not, the reduction is decorative.

## 8. What This Changes In The Skill

The skill should stop asking first:

- what chapter is this?
- what school stage is this?
- what standard routine applies?

It should ask first:

- is this problem truly high-level, or only high-surface?
- what is the lowest honest layer that still preserves the controller?
- can the route be rebuilt there without cheating?

## 9. Final Rule

Do not worship the high form.
Do not flatten it carelessly either.

Instead:

- recover the shared controller
- return to the lowest honest layer
- solve there if lawful
- climb back only when necessary
