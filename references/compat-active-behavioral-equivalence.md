# Compat-Active Behavioral Equivalence

This file defines the compatibility-active form of behavioral convergence toward the ideal layer.

Its purpose is:

- make compat-only execution explicitly aim at behavior-level convergence
- avoid mistaking textual resemblance for real progress

## Core Claim

Compat-only execution gets closer to the ideal layer when:

- more of the route feels internally modulated
- more of the route stays small without visible control
- more of the route finishes through silent coordination of fields

It does not get closer by:

- adding more named layers
- adding more visible structure
- sounding more like the ideal docs

## What This Means In Compatibility Form

At the compatibility-active level, behavioral equivalence means:

- pressure fields matter more than wording
- route shape matters more than explicit doctrine
- convergence is judged by smaller, more stable, more self-propelled routes

This is not:

- perfect doc equivalence
- full native indistinguishability

But it is already more than:

- static contract compatibility

## Allowed Compatibility Pressure

Compat-only execution may already preserve:

- field modulation
- unresolved-pressure carryover
- target-complete carrier pressure
- direct-readout readiness
- anti-regrowth restraint

These are enough to support:

- weak behavior-level convergence

without:

- visible workflowification

## What Must Stay Forbidden

Do not turn behavioral equivalence into:

- “imitate the ideal docs”
- a checklist of ideal-layer phrases
- more explicit process control
