# Wu Boshi Series-Mined Method Clusters

This file records a key lesson from public video series:

- the real asset is not one solved problem
- the real asset is the repeated method cluster behind a whole series

## Core Principle

When a creator repeatedly explains:

- seven triangles
- ruler comparison
- same mother-function families
- derivative question types
- over-standard functions
- guess-then-verify answers

the library should not memorize the episode list.
It should extract the stable cluster:

- what family keeps recurring?
- what burden keeps being killed?
- what recognition cue keeps triggering the same primitive?

## What A Series Usually Contains

A strong public series often bundles:

1. a shell-kill slogan
2. a repeated prototype family
3. a recognition cue
4. a cheap decisive move
5. a transfer promise across many problems

That makes a series a method mine, not just a playlist.

## Common Series-Level Clusters

### 1. Prototype Cluster

Examples:

- seven triangles
- mother-function dictionaries
- over-standard function banks

What to extract:

- prototype recognition
- family routing
- normal-form thinking

### 2. Fast Comparison Cluster

Examples:

- ruler method
- compare-size one-step answers

What to extract:

- comparison axis
- visual proxy
- threshold separation
- read-not-solve instinct

### 2.5 Symmetry Cluster

Examples:

- hidden mirror or center-balance inside comparison and geometry routes
- relabel symmetry behind direct counting and matching routes

What to extract:

- duplicate-work collapse
- balanced-frame recognition
- symmetry-supported route guess

### 3. Guess-Then-Verify Cluster

Examples:

- “高考题可以直接出答案，先猜后证”

What to extract:

- route guess first
- answer-shape pressure
- smallest trust bill afterward
- baited-route refusal

### 3.5 Special-Value Cluster

Examples:

- special parameter values
- convenient representatives
- midpoint / boundary / symmetry-value probes

What to extract:

- structurally dangerous test values
- landmark-value elimination
- canonical representative selection

### 4. Topic Compression Cluster

Examples:

- “导数题全在这五个类型里”
- “13个超纲函数”
- “十大同构母函数”

What to extract:

- topic-to-family compression
- family-level attack before local derivation
- reusable routing maps

## Extraction Rule

When mining a series, do not store:

- only episode titles
- only source links
- only one-off tricks

Store:

- cluster name
- recurring primitive
- recurring shell-kill
- recurring transfer promise
- common failure mode

## Hard Rule

A video series should enrich the library at the family or primitive-cluster level.
If it only adds more solved instances but no stronger cluster, the mining was too shallow.
