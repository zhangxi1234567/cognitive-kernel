# Wu Boshi Generalization Rules

This file defines the real objective of the skill:

not to quote Wu Boshi,
but to solve new problems the way his public method families suggest.

The scope is not limited to school math framing.
It should generalize across:

- math
- physics
- chemistry
- low-complexity to research-style problems

## Core Principle

Every named public method should be converted into a reusable abstract move.

Every abstract move should also support:

- shell-killing
- anti-regrowth
- target-only solving when legal
- aggregate-first control when available
- answer-copy barrier breaking

Do not ask:

- "Is this exactly a seven-triangles problem?"

Ask:

- "What is the deeper move behind seven triangles, and can I reuse that move here?"
- "What ordinary route is being baited here, and what smaller route can replace it?"

## The Abstract Moves

### 1. Externalize

If the problem is trapped in symbols, words, or mental load:

- draw it
- map it
- table it
- label it
- turn it into motion or shape

### 2. Compress

If the topic feels infinite:

- compress it into a few families
- name the families
- route the current problem into one family

### 3. Compare

If the task looks computational:

- ask whether it is really a comparison problem
- compare before solving

### 4. Count

If the structure repeats:

- count before deriving
- partition before computing

### 5. Anchor

If the picture is unstable:

- find a few anchor points
- reason from them

### 6. Reframe

If the learner is intimidated:

- strip prestige language
- rename the task in blunt everyday language
- make the fake complexity visible

### 7. Find the Deep Law

If the surface trick changes across topics, search for the deeper law that remains:

- invariant
- conservation
- symmetry
- scaling
- limiting case
- structural analogy

## Complexity-Downgrade Rule

This skill must aggressively search for an earlier honest layer or a lower-complexity solution.

That means:

- formal problem -> try structure-reading first
- structure-heavy problem -> try toy-model intuition first
- hard problem -> try the earliest honest layer first
- any domain problem -> try the lower-complexity analogue first

Do not show off a heavier method if a lower-complexity method is clearer and faster.

Even for highly formal or research-style problems, first expose the lowest truthful structure layer underneath the formal machinery.

## Fast-Solve Rule

The default target is:

- lowest cognitive load
- shortest stable path
- clearest transferable insight

Only move to formal derivation when:

- the shortcut is ambiguous
- the user asks for rigor
- the domain is high-stakes

## Low-Complexity Routing

For low-complexity layers and early-structure versions, prefer:

- number sense
- ratio
- area intuition
- similar-shape intuition
- symmetry
- parity
- case splitting
- table-making
- plug small numbers
- work backward
- compare instead of compute

For physics and chemistry, prefer:

- balance before formula
- particles before symbols
- geometry before algebra when possible
- energy/accounting before memorized rules

Do not make low-complexity problems look harder than they are.

## Signs The Skill Is Working

The answer should feel like:

- "This can be done from a low-complexity structural reading."
- "I do not need the heavy method."
- "The problem is just a picture/comparison/count."
- "I can reuse this move on other questions."

## Signs The Skill Is Failing

- using advanced machinery too early
- repeating method labels without solving the new problem
- sounding impressive instead of clear
- giving long algebra when one observation was enough
- failing to drop the problem to an earlier honest layer

## Operational Prompting Rule

When using this skill, silently run this checklist:

1. What is the earliest honest version of this problem?
2. What is the picture?
3. What can be compared?
4. What can be counted?
5. What family does this belong to?
6. What is the one move that kills most of the difficulty?
7. Can I explain it so someone operating one complexity layer lower would still feel smart?
8. What deep law survives if I change the subject from math to physics to chemistry?
