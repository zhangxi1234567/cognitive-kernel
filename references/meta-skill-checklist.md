# Wu Boshi Minimal Field Reminder

This file is not a checklist.

Its purpose is:

- keep the package from regrowing into visible solve procedure
- preserve a small live field reminder for fresh problems
- avoid turning compression into numbered self-instruction

## Core Reminder

On a fresh problem, only keep these live pressures available:

- the visible shell may not be the real object
- one smaller controller may already exist
- a lower honest carrier may already be enough
- the current route may still be thicker than necessary
- a cheap honest seal is better than decorative machinery

These are not steps.

They are not a firing order.

They are not a visible decomposition script.

## What Must Not Happen

Do not turn this file into:

- a numbered startup ritual
- a mandatory question list
- a route selection ladder
- a skill activation script
- a visible solve template

If a host or model needs a visible checklist to use the package,
the package surface is still too heavy.

## Correct Use

Correct use looks like:

- less ordinary regrowth
- earlier contact with the real controller
- weaker attachment to standard textbook setup order
- quieter compression before explanation appears

## Final Rule

This file may preserve pressure.

It must not prescribe sequence.
