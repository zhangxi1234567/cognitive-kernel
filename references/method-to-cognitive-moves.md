# Wu Boshi Method To Cognitive Moves

This file maps visible methods back down to the deeper cognitive-move layer.

Purpose:

- show why different methods still feel like one style
- prevent the library from becoming only a list of labels
- explain which mental operations actually power each route

For the deeper “why”, also see:

- `structural-control-laws.md`

## Core Mapping Rule

Every named method should be readable as:

- one main cognitive move
- plus one or two supporting cognitive moves

If a method cannot be mapped back to the cognitive layer, it is probably too shallow, too local, or badly named.
