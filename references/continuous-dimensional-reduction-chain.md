# Wu Boshi Continuous Dimensional Reduction Chain

This file defines the hidden shape of the strongest routes in the project.

The best routes are often not one magic trick.

They are:

- a continuous chain of reductions

That means:

- one reduction reveals a smaller carrier
- that carrier reveals another reduction
- that reduction reveals an even smaller carrier
- until the target is almost directly readable

## 1. Core Preference

The strongest routes often behave like:

- continuous dimensional-reduction chains

not:

- one reduction
- then ordinary solving

## 2. Typical Chain Shape

A strong chain may look like:

- shell -> relation
- relation -> one parameter
- one parameter -> direct target expression
- direct target expression -> minimal seal

or:

- surface -> line
- line -> point relation
- point relation -> direct readout

or:

- point coordinates -> displacement parameter
- displacement parameter -> determinant form
- determinant form -> area direct readout

This usually works best when:

- the first jump already landed on a strong main carrier
- and later steps did not abandon that carrier

Use:

- `first-jump-main-carrier-selection.md`
- `stay-on-the-smallest-carrier.md`

to strengthen those two moments.

## 3. Why This Matters

If the route stops after the first reduction, the problem often remains larger than necessary.

If the route keeps reducing, then:

- many objects die
- many variables vanish
- derivation turns into readout
- the route starts to feel obvious in hindsight

That is usually what the user recognizes as real `降维打击`.

## 4. Evaluation Question

For any successful route, ask:

- how many real reductions happened before the final seal?

If the answer is only one, the route may still be improvable.

If several reductions happened in one coherent chain, the route is closer to the project's ideal.

## 5. Final Preference

The strongest routes do not merely find the right idea.

They keep reducing until the answer becomes small.

Use:

- `meta-preferences-not-fixed-rules.md`

to read this as a route ideal, not a mandatory visible pattern.
