# Wu Boshi Assignment As Mechanism Probe

This file defines assignment as a bottom-layer probing logic.

Its purpose is:

- stop treating assignment as a cheap trick
- treat chosen values as stress tests on the hidden mechanism

## Core Rule

Assignment is lawful when the chosen value is not random.

Ask:

- what value kills the most fake freedom?
- what value reveals symmetry, zero structure, boundary behavior, or common parameter control?

Then compress:

- abstract structure -> stressed probe case

## Bottom-Layer Logic

- a good assignment is a diagnostic load on the mechanism
- it is chosen because it removes slack or isolates one controller

This is why:

- `0`
- `1`
- `-1`
- equal variables
- boundary values
- symmetric points

show up so often:

- they are low-cost mechanism revealer values

## What This Explains

- special value probing
- puzzle assignment
- function-value testing
- parameter screening

## Failure Boundary

Assignment fails when:

- it is chosen only because it is easy
- it does not stress the real structure
- it kills too much and changes the mechanism instead of exposing it

## Transfer

- school math: plug easy values
- university math: choose basis vectors, canonical elements, extremal examples
- research-style transfer: pick the smallest experiment that stresses the live mechanism
