# Primitive: Definition As Direct Readout

## Pattern

The problem statement already contains a defining relation that directly controls the target.
Standard machinery would only rebuild what the definition already gives.

## Trigger Signal

- the object is introduced by a definition rather than by a ready-made formula
- the target is visibly close to the defining relation
- the standard route would introduce coordinates, derivatives, or heavy algebra before spending the definition
- the statement contains words like distance difference, equal ratio, functional equation, periodic shift, favorable over total, focus/directrix, or tangent definition
- auxiliary machinery feels like rebuilding the bridge instead of crossing it

## Level-Agnostic Core

Do not treat the definition as background.
Treat it as the shortest readout of the controlling relation.

## Control Mechanism

The move works because:

- the defining relation already compresses the object into a target-relevant law
- the definition preserves the exact admissibility conditions that later machinery would otherwise reconstruct
- the target often depends on the relation named by the definition, not on the full object

## Preserved Structure

Typical preserved items:

- the exact defining relation
- target-relevant admissibility
- direct relation between object and asked quantity
- map from definition language to the target without importing extra objects

## Earliest Honest Layer

Before solving, ask:

- what does the definition literally say?
- if I draw or restate only that relation, does the target already become visible?
- what burden disappears if I refuse to reconstruct the whole object first?

## Fast Move

1. Rewrite the definition in the problem's own variables or picture.
2. Ask what the definition already controls.
3. Read the target through that relation first.
4. Add secondary machinery only if the definition alone cannot close the route.

## Compression Move

Typical burden deletion:

- object reconstruction -> relation readout
- chapter routine -> definition picture
- heavy auxiliary setup -> direct use of the given law
- formal shell -> primitive relation

## Level Transfer

- low-complexity form: use the rule exactly as stated
- compact formal form: definition becomes the first equation or picture, not the last justification
- advanced formal form: derive through the governing relation before launching full machinery

## Cross-Domain Analogue

- debugging: use the failing contract/interface definition before tracing every downstream symptom
- product/strategy: use the explicit success metric definition before modeling secondary effects
- science: use the defining conservation or constitutive law before adding full formal apparatus

## Where It Transfers

- conics via focus/directrix or focal definitions
- functional equations
- periodicity and symmetry-by-definition problems
- probability once a valid uniform encoding is established
- geometry or algebra questions where the target sits directly on the defining relation

## Failure Mode

- the definition is necessary but not sufficient for the target
- the target depends on extra structure not carried by the bare definition
- the definition is used too literally and branch/admissibility details are lost
- the defining relation gives a route hint but not the whole controlling mechanism

## Verification Bill

State:

- which part of the definition is doing the work
- why that relation already controls the target
- what extra condition, if any, still needs a minimal seal

## Seed Examples

- conic questions solved directly from focus/directrix or distance-difference definition
- function problems solved from the given functional equation before derivative machinery
- periodicity or symmetry read directly from the defining shift relation
- probability/count problems solved from the valid favorable/total encoding once uniformity is justified
