# Wu Boshi Deep Logic

This file defines the deepest layer of the skill.

The public Wu Boshi methods are examples.
The real target is the underlying problem-solving logic beneath them.

## The Thesis

Most hard problems become easier when you stop treating them as chapter-specific techniques and start treating them as instances of a few deep structures.

The skill should learn those structures and reuse them everywhere:

- low-complexity intuition
- structure-reading layer
- compact derivation layer
- formal advanced layer
- research-style reasoning
- mathematics
- physics
- chemistry

## Deep Structures

### 1. Invariant

Ask:

- what stays the same while other things change?
- what quantity, relation, or pattern survives the transformation?

Use in:

- algebra
- geometry
- mechanics
- equilibrium
- reaction constraints
- proof problems

### 2. Conservation

Ask:

- what is conserved?
- what only moves from one place to another?
- what bookkeeping quantity must balance?

Use in:

- physics: energy, momentum, charge
- chemistry: mass, atoms, charge, electron count
- math: measure, parity, total count, conserved combinatorial quantity

### 3. Symmetry

Ask:

- what does not change under swapping, reflection, rotation, translation, relabeling?
- where can symmetry remove work?

Use in:

- geometry
- functions
- physics
- molecular structure
- abstract algebraic reasoning

### 4. Scaling

Ask:

- if I enlarge, shrink, or rescale the system, what changes and what stays proportional?
- what dominates in the large or small limit?

Use in:

- functions
- asymptotics
- mechanics
- field problems
- chemistry concentrations and rates

### 5. Limiting Case

Ask:

- what happens at the edge?
- what happens when a parameter becomes 0, 1, infinity, equal, or extreme?

Use in:

- inequalities
- derivatives
- physical intuition
- equilibrium and rate arguments
- sanity checks

### 6. Structural Analogy

Ask:

- what simpler problem has the same skeleton?
- is this "really" a comparison, count, balance, flow, geometry, or template-recognition problem?

This is the main bridge across disciplines.

### 7. Representation Switch

Ask:

- should this be words, picture, table, graph, particles, forces, geometry, or algebra?

Many hard problems stay hard only because they are stuck in the wrong representation.

## Complexity-Layer Rule

No matter how advanced the problem is, search for the earliest honest layer at which its skeleton can still be seen.

Examples:

- advanced analysis -> local picture, monotonicity, bounding, limiting case
- abstract algebra -> symmetry, operation pattern, invariant structure
- quantum or statistical arguments -> counting states, symmetry, conservation, limiting case
- organic chemistry mechanism -> electron flow bookkeeping and structural constraints
- cross-domain transfer -> conserved skeleton, not vocabulary match

Do not confuse advanced notation with advanced core logic.

## Universal Checklist

Before solving, ask:

1. What is being conserved?
2. What is invariant?
3. What is symmetric?
4. What happens in the limit?
5. What can be compared instead of solved exactly?
6. What can be counted instead of derived?
7. What is the simplest analogous problem?
8. What is the best representation?
9. Can this be dropped by one honest complexity layer?
10. What one observation kills most of the difficulty?

## Success Condition

The skill is working if it can:

- solve unfamiliar problems using familiar deep structures
- move between math, physics, and chemistry without losing clarity
- explain advanced ideas in low-load language
- preserve truth while cutting unnecessary formal weight
