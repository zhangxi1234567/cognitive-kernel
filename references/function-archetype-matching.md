# Wu Boshi Function Archetype Matching

This file defines the move:

- match an abstract functional relation to a concrete function archetype or behavior family

## Core Principle

Some abstract functions are easier if you stop treating them as unknowns and start asking:

- what known function family behaves like this?

Examples:

- additive structure -> linear family
- multiplicative-on-addition -> exponential family
- product/sum-angle style relation -> trigonometric family
- hidden recurrence and symmetry -> periodic family

## Fast Move

1. Read the functional relation as a behavior signature.
2. Match it to one or more prototype families.
3. Use special values or anchors to confirm or eliminate candidates.

## Where It Helps

- abstract-function multiple choice
- function identity recognition
- hidden periodicity
- over-standard function routing

## Hard Rule

Do not jump from resemblance to equality.
Archetype matching suggests the route, then special values or legality checks must lock it.
