# Wu Boshi Anti-Freeze Rule

This file prevents the skill from hard-freezing into a fixed trick catalog.

Its purpose is:

- keep the skill generative
- stop good primitives from turning into rigid templates
- preserve cross-domain transfer into research, debugging, strategy, and other non-school surfaces

## Core Rule

The skill may stabilize:

- quality gates
- mechanism standards
- anti-regression checks
- routing priorities

But it must not hard-freeze:

- one canonical variable choice
- one canonical first step
- one canonical representation
- one canonical wording of the essence
- one chapter-bound trigger list

Rule:

- fix the mechanism
- do not fix the costume

## What Must Stay Flexible

For the same underlying primitive, different surfaces may require different:

- variables
- diagrams
- readouts
- toy models
- sanity checks
- experimental probes
- synthesis sentences

Examples:

- in school math, `projection readout` may become height / gap / section
- in physics, it may become one axis, one mode, or one conserved component
- in research, it may become one assay, one latent factor, or one decisive observable
- in debugging, it may become one failing transition or one critical log line

The primitive is the same.
The representation must be regenerated.

## Freeze Failure Modes

### 1. Trigger Freeze

Shape:

- the model learns one visible wording and stops calling the primitive when the wording changes

Why it fails:

- chapter memory replaced mechanism recognition

### 2. Variable Freeze

Shape:

- the model always introduces the same textbook variables even after identifying a smaller object

Why it fails:

- the essence did not control the route

### 3. Representation Freeze

Shape:

- the model always redraws the problem in the same surface form
- even when another representation is cheaper in the current domain

Why it fails:

- the model memorized a shell, not a reduction move

### 4. Example Freeze

Shape:

- one beautiful seed example becomes the default route everywhere

Why it fails:

- transfer collapses outside the seed surface

## Allowed Stability

The following should be stable:

- ask for the essence before choosing variables
- test whether the essence changed the route
- prefer the smallest truthful controller
- verify the fragile hinge cheaply
- expose preserved structure and failure boundary

These are stable meta-rules, not frozen local tactics.

## Regeneration Rule

Each time the surface changes, regenerate:

1. what the true object is
2. what the controlling readout is
3. what representation is cheapest now
4. what verification is cheapest now

Do not import these unchanged from an older example unless the present surface truly preserves them.

## Hard Rule

If a primitive only works when its old:

- variable names
- picture style
- chapter label
- example shell

are preserved, then the primitive has not been learned deeply enough.
