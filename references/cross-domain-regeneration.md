# Wu Boshi Cross-Domain Regeneration

This file defines how one primitive should regenerate across domains without freezing into one surface.

Its purpose is:

- make transfer operational
- preserve deep mechanism while allowing new local forms
- support movement from mathematics into research-style reasoning without forcing math-shaped outputs everywhere

## Core Rule

Transfer is not copy-paste.

Transfer is:

- same control mechanism
- new surface object
- newly regenerated representation
- newly regenerated verification

If only the wording changes but the representation and verification should also have changed, the transfer is incomplete.

## Regeneration Questions

When moving into a new domain, ask:

1. what plays the role of the object now?
2. what plays the role of the target now?
3. what is the preserved structure?
4. what is the cheapest controller or readout in this domain?
5. what is the domain-appropriate cheap verification?

## Example: Projection Readout

Math surface:

- area -> height
- volume -> section
- many variables -> one axis

Research surface:

- many measurements -> one decisive observable
- many parameters -> one effective control factor
- many pathways -> one rate-limiting step

Debugging surface:

- many logs -> one first divergence point
- many services -> one critical path bottleneck

## Example: Local Seam Controls Global

Math surface:

- boundary point
- smallest dangerous case
- first overlap / omission

Research surface:

- instability onset
- threshold crossing
- first falsifying regime

Debugging surface:

- first broken transition
- first state mismatch
- smallest failing reproducer

## Example: Cheapest Honest Seal

Math surface:

- one boundary check
- one monotonicity hinge
- one plug-back

Research surface:

- one separating experiment
- one measurement that kills a hypothesis family
- one stress test at the decisive regime

Strategy surface:

- one metric that separates options
- one scenario test that kills a bad plan

## Hard Rule

The skill must not insist that:

- every domain transfer still look like a math solution
- every target be expressed as a symbolic derivation
- every verification be a proof-style seal

Cross-domain transfer succeeds when the same mechanism survives with a domain-native representation and a domain-native trust bill.
