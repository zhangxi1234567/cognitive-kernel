# Primitive: Canonical Normalization

## Pattern

A decorated object becomes easy once it is rewritten as a cleaner mother object or normal form.

## Trigger Signal

- the object looks hard mainly because of coordinates, weights, parameters, or surface decoration
- a standard mother object is visibly nearby
- the target seems to depend on structure more than raw presentation
- a scaling, frame change, symmetry choice, or dimensionless rewrite feels natural
- the current representation is carrying too much fake burden

## Level-Agnostic Core

The same structure hides behind many coordinate choices, weights, or surface decorations.
Normalize the representation before solving.

## Control Mechanism

The move works because the target-relevant structure survives the rewrite:

- invariance
- equivalence under change of frame
- scaling law
- canonical representative
- same control relation in cleaner coordinates

## Preserved Structure

Typical preserved items:

- incidence or order structure
- target-relevant ratios
- monotone orientation
- algebraic type / family identity
- qualitative behavior inherited from the mother object
- map-back law for the asked quantity

## Earliest Honest Layer

Make the ugly thing look like the old familiar thing first.
Do not fight the decoration directly.

## Fast Move

1. Name the mother object.
2. Rewrite the current object into that cleaner form.
3. Solve in the normalized world.
4. Map back only the target quantity that the problem asked for.

## Compression Move

Typical burden deletion:

- ugly object -> mother object
- many coefficients -> one standard form
- distorted geometry -> canonical geometry
- dimensional clutter -> dimensionless control relation
- local novelty -> inherited prototype behavior

## Level Transfer

- low-complexity form: ugly shape -> familiar shape
- compact formal form: normalize variables / coordinates / scale
- advanced formal form: canonical form, diagonalization, quotient representative, dimensionless reduction

## Cross-Domain Analogue

- physics: turn a system into dimensionless form
- debugging: reduce many symptoms to a minimal reproducer
- product thinking: reduce noisy situations to one canonical decision frame

## Where It Transfers

- conics and analytic geometry
- function transformation problems
- linear algebra and quadratic forms
- asymptotic and perturbative models
- optimization and modeling

## Failure Mode

- the normalization does not preserve the asked quantity
- the map-back distortion is ignored
- the mother object match is cosmetic rather than structural
- the rewrite simplifies notation but not control

## Verification Bill

State:

- what is preserved
- what changes under normalization
- how the target is translated back

## Seed Examples

- ellipse / conic reductions to cleaner prototype geometry
- normalized function / mother-function routing
- quadratic form -> normal form style reductions

This primitive should activate across visibly different surfaces:

- ellipse or conic -> cleaner prototype geometry
- transformed function -> mother function / base curve
- quadratic form -> normal form
- physical model -> dimensionless reduced model
- noisy real system -> minimal reproducer or canonical failure case
