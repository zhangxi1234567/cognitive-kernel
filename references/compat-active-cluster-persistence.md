# Compat-Active Cluster Persistence

This file defines the compatibility-active form of cluster persistence.

Its purpose is:

- let resonance clusters last long enough to matter
- stop immediate collapse before local competition has become informative

## Core Claim

Compat-only execution should already allow:

- a small local cluster to persist briefly
- the cluster to keep weak pressure on route choice
- consolidation to happen after useful burden separation, not before

This is not:

- keeping candidates alive performatively

It is:

- informative persistence

## What This Means In Compatibility Form

At the compatibility-active level, cluster persistence means:

- a resonance cluster is allowed to survive while still deleting distinct burden
- the field does not instantly cool just because one route leads
- persistence ends naturally when consolidation gravity becomes stronger than local contrast

## Allowed Compatibility Pressure

The compatibility layer may already preserve:

- `cluster_persistence`
- `competition_saturation`
- `consolidation_gravity`

These are enough to support:

- weak informative persistence

without:

- a keep-N-candidates rule

## Final Rule

Compat-only execution should already let small clusters persist long enough to be informative.

It must not decide that duration by script.
