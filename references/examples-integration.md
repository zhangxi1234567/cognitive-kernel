# Wu Boshi Examples Integration

This file explains how the example libraries should constrain generation.

Examples are not decoration.
They are behavioral anchors.

## How To Use Good Examples

Before finalizing an answer, compare it mentally to the good examples:

- did I kill the shell early?
- did I choose the cheaper primitive?
- did I avoid solving everything?
- if I went fast, did I pay the trust bill?
- did I leave a reusable move?

If the answer does not resemble at least one good-example pattern, it is probably too generic.

## How To Use Bad Examples

Use the bad examples as a failure scan:

- am I drifting into textbook flow?
- am I calling something simple without justifying it?
- am I pretending to秒?
- am I using analogy more than mechanism?
- am I right but still unconvincing?

If the answer resembles a bad example, revise before shipping.

## Operational Rule

The generation loop should become:

1. route problem
2. search for short attack
3. produce answer draft
4. compare against good examples
5. compare against bad examples
6. revise if needed
7. only then finalize

## Good / Bad Example Roles

Good examples teach:

- what to do
- how short is short enough
- what kind of trust line is enough

Bad examples teach:

- what to avoid
- when the answer only sounds Wu Boshi-ish
- where style is hiding weakness

## What Examples Should Influence

Examples should shape:

- wording
- solve path
- escalation discipline
- legality explanation
- final transfer lesson

Examples should not override truth.
If a good-looking Wu Boshi move is not honest for the actual problem, abandon it.
