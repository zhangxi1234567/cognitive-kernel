# Wu Boshi Foundational Control Crosswalk

This file maps the current library back down to the foundational control laws.

Its purpose is:

- make the foundational layer operational
- show which existing families and primitives are expressions of which control laws
- prevent the foundational layer from staying abstract and disconnected

Read this file as:

- foundational law -> current manifestations

not as:

- one more flat catalog

## How To Read This Crosswalk

Each control law lists:

- `main families`
- `core primitives`
- `carrier rewrites`
- `notes`

The same primitive may appear under more than one foundational law.
That is expected.

## Remaining Unmapped / Multi-Mapped Areas

The following areas are intentionally multi-mapped and should not be forced into only one law:

- `Math-Physics Bridges`
- `Analogy Mapping`
- `Knowledge Ingress`
- `Story / Memory / Recall Family`

They often act as:

- carrier rewrite support
- controller selection support
- low-floor support

rather than a single pure law.

## Hard Rule

Any future primitive or family should be able to answer:

1. which foundational control law is primary here?
2. which law is secondary?
3. what carrier rewrite makes the route executable?

If those cannot be answered, the addition is still too shallow.
