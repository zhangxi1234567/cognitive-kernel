# Anti-Overdefinition Of Essence

This file defines a hard rule:

- the project must not over-explain the agent's internal descent toward essence

Its purpose is:

- preserve self-driven exploration
- prevent the package from teaching a new hidden workflow under the name of depth

## Core Claim

The project should not externally define too precisely:

- what exact internal questions the model must ask
- how many layers it must peel
- which shell types it must name
- what order internal decomposition must follow

Those moves risk turning:

- internal discovery

into:

- external choreography

## What Must Not Be Done

Do not write package instructions that say things like:

- first ask this exact why-question
- then peel this exact outer shell
- then check this exact inner shell
- then recurse again

Do not freeze a taxonomy of:

- shell types
- inner layers
- essence stages
- decomposition ladders

as if the model should replay them literally.

## What The Package May Still Do

The package may:

- bias the model toward smaller truthful structure
- make middle-layer stopping feel unsatisfying
- preserve competition between candidate attack angles
- preserve pressure toward shared controllers and lower carriers

But it must leave the actual exploratory motion:

- internal
- provisional
- problem-shaped

## Why This Matters

If the package over-defines internal essence descent, then even a deep-looking system becomes:

- an externalized thought script

The model may then:

- sound recursive
- sound deep
- sound principled

while no longer truly exploring.

That is exactly what this project must avoid.

## Good Packaging

Good packaging says:

- smaller truthful structure matters
- one route should not stop at a fake middle layer
- internal competition should stay alive
- external control should stay weak

Good packaging does not say:

- here is the exact inner interrogation sequence to run every time

## Relation To Instinct

Use:

- `dimensional-reduction-instinct.md`

for the positive target:

- the model should increasingly feel when the current layer is still not fundamental enough

This file adds the negative constraint:

- do not replace that feeling with a package-defined decomposition ritual

## Relation To Compatibility

Use:

- `non-workflow-compatibility-contract.md`
- `compat-layer-vs-native-runtime.md`

This file sharpens them further:

- compatibility must not only avoid workflow
- it must also avoid over-defining the model's internal essence descent

## Final Rule

The package should teach pressure toward essence.

It must not teach a mandatory script for how essence must be reached.
