# Wu Boshi Periodicity As Compression

This file defines how periodicity acts as a reduction move.

Its purpose is:

- turn infinite-looking behavior into one cycle
- replace long-range tracking with one repeating cell
- treat periodicity as a controller, not only as a property

## Core Rule

When a structure is periodic, do not keep solving globally.

Ask:

- what is one full cycle?
- what changes inside the cycle?
- what survives from cycle to cycle?

Then compress:

- global object -> one period block

## Trigger Signals

- repeated trig values
- translated graphs that recur
- recurrence behavior with fixed cycle length
- parity plus shift giving repetition
- modular or residue-like repeating structure

## Compression Moves

- many terms -> one cycle pattern
- infinite family -> finite pattern bank
- global sum/product/count -> per-cycle readout plus remainder
- function behavior -> one standard window

## Why It Works

- periodicity kills fake infinity
- one period is the true mother object
- all later work is repetition bookkeeping

## Good Uses

- trigonometric sequences
- transformed function behavior
- repeated sign / parity patterns
- modular-style counting

## Failure Boundary

Do not overfire this move when:

- the behavior is only locally repetitive
- the cycle length is not actually stable
- amplitude / shape still changes between cycles

## Transfer

- school math: repeated trig and sequence values
- university math: periodic functions, residues, quotient behavior
- research-style transfer: one cell, one orbit, one recurring regime
