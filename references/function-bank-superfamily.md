# Wu Boshi Function-Bank Superfamily

This file is the consolidated superfamily surface for the function-bank line.

Its purpose is:

- unify mother-function dictionaries, over-standard function banks, and function-archetype routing
- separate stable bank logic from unresolved public member lists

## Core Principle

Do not treat each function as new.

Ask:

- what behavior bank owns this object?
- what mother form is hiding inside it?
- what default route should activate first?

## The Three Internal Layers

### 1. Archetype Recognition Layer

Use:

- `Function Archetype Matching`
- `Mother-Pattern Recognition`

Purpose:

- identify the behavior family
- strip novelty from the surface form

### 2. Routing Layer

Use:

- `Topic Routing Maps`
- `Function-Bank Clusters`

Purpose:

- compress a large topic into a small number of route families
- decide which primitive or family should fire first

### 3. Public-Bank Registry Layer

Use:

- `public-function-bank-registry.md`

Purpose:

- store unresolved but confirmed public finite banks such as:
  - `13个超纲函数`
  - `十大同构母函数`
  - `六大完美超越函数`

## What This Superfamily Already Knows

- how to recognize mother behavior
- how to route novelty into old families
- how to distinguish stable bank logic from missing member lists

## What Still Depends On More Source Access

- exact public member enumerations
- member-by-member mapping into primitives
- transcript-level definitions of the public named banks

## Hard Rule

Do not confuse:

- “we do not yet know every public bank member”

with:

- “the bank logic is not integrated”

The bank logic is already part of the library.
