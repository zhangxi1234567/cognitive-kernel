# Wu Boshi Reasonableness And Dimension Logic

This file explains the deeper logic behind dimensional, scale, and plausibility checks.

It answers:

- why can unit, scale, and magnitude reject bad routes so early?
- why is this not “intuition only”, but a real structural filter?

## Core Principle

A route can fail before proof if it already violates the object's native scale or type.

That failure may show up as:

- wrong unit
- impossible size
- impossible probability
- impossible geometry
- wrong growth order

This is not weak intuition.
It is structural mismatch.

## Why It Works

Every valid answer must live inside a compatibility envelope:

- type compatibility
- scale compatibility
- boundary compatibility

If a candidate leaves that envelope, it dies before any detailed proof is needed.

## The Three Logic Layers

### 1. Type Compatibility

- like must combine with like
- a length cannot be added to an area without a bridge
- a probability cannot exceed its sample-space logic

### 2. Scale Compatibility

- a value that is two orders too large for the geometry is already wrong
- a quantity that exceeds the picture's capacity is already wrong

### 3. Boundary Compatibility

- if the route contradicts edge cases or special positions, it is structurally wrong even before full proof

## Why This Is Cross-Domain

This law works across:

- arithmetic and algebra
- geometry
- probability
- calculus and growth-rate comparison
- physics units and scales
- chemistry conservation and ratio sanity
- debugging and systems plausibility

## Hard Rule

Reasonableness is not a substitute for proof.
It is a first structural filter:

- keep plausible routes alive
- kill impossible ones early
