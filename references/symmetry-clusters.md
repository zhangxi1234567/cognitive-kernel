# Wu Boshi Symmetry Clusters

This file defines symmetry as a recurring public-series method cluster.

## Core Principle

Symmetry is not a local geometry trick.
It is a universal burden-killer.

It can:

- remove duplicate cases
- collapse two objects into one summary
- expose a fixed point, midpoint, or balanced frame
- turn a comparison into one shared axis
- justify a guess before the smallest trust bill

## Public-Series Role

Across public teaching lines, symmetry is often not named as the headline method,
but it repeatedly powers:

- seven-triangle recognition
- compare-size shortcuts
- direct-count compressions
- graph and function shape reading
- fixed-object and midpoint models

This makes symmetry a hidden backbone cluster, not only a local supporting trick.

## Common Symmetry Subclusters

### 1. Mirror / Center Symmetry

Use when:

- left/right, above/below, or before/after roles match
- a center, midpoint, or reflection stabilizes the picture

### 2. Relabel Symmetry

Use when:

- objects are interchangeable
- labels create fake extra work

### 3. Balanced-Frame Symmetry

Use when:

- one middle point, average, or equal-offset frame exposes the control object

### 4. Symmetry-As-Guess Support

Use when:

- the final answer looks too clean to come from asymmetric chaos
- a symmetry can pressure the route or answer shape before full proof

### 5. Symmetry-To-Periodicity

Use when:

- one symmetry axis or center appears in a function problem
- the real payoff is not only symmetry itself, but a forced repeated structure

## What Symmetry Usually Replaces

- repeated casework
- two-sided calculations
- arbitrary label tracking
- full reconstruction of mirrored objects
- large-index brute force once a period is actually available

## Hard Rule

Do not use symmetry only as a slogan.
State:

- what is symmetric
- what work disappears because of that symmetry
- what still needs checking because symmetry does **not** decide it alone
