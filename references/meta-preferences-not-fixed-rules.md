# Wu Boshi Meta-Preferences, Not Fixed Rules

This file protects the project from another subtle drift:

- turning strong thinking preferences into rigid procedural rules

The project is not trying to install:

- one permanent solve script

It is trying to cultivate:

- durable thinking biases
- controller-seeking habits
- compression-seeking instincts
- honesty-preserving route preferences

Short version:

- direction, not script

## 1. Core Principle

Files in this library often say things like:

- prefer
- bias toward
- keep asking
- do not stop until
- test whether

These should be read as:

- meta-preferences
- cognitive tendencies
- route-selection pressure

not as:

- mandatory visible step order
- frozen global SOP
- chapter-style checklist

## 2. Why This Matters

If the project hardens into fixed procedure, it may help on familiar tasks but it will damage:

- discovery
- scientific exploration
- unfamiliar-problem adaptation
- genuine first-principles thinking

Research-grade usefulness requires:

- freedom to notice new structure
- freedom to abandon a bad framing
- freedom to invent a new carrier
- freedom to compress in an unfamiliar way

The library should strengthen these freedoms, not replace them with compliance.

## 3. The Right Reading

When the project says:

- prefer controller-first
- prefer shorter surviving routes
- keep compressing after the first insight
- use multi-route search when needed
- favor one-route elite execution when possible

the meaning is:

- push thought in this direction

The meaning is not:

- always do this visibly in this exact order

## 4. Final Rule

The project should shape instinct, not trap it.
