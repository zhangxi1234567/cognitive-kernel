# Seam Ranking

This file defines the project's positive account of seam ranking.

Its purpose is:

- distinguish multiple locally visible seams without turning them into a probe sequence
- preserve sensitivity to which seam is structurally most dangerous
- prevent seam-finding from becoming a checklist of standard substitutions

## Core Claim

Seam ranking is not:

- “try seam A, then seam B, then seam C”

Seam ranking is:

- local discrimination between seams by how much real burden they are likely to delete

## Stronger Seam

A seam is locally stronger when it is more likely to:

- collapse two branches into one carrier
- reveal a lower controller directly
- kill a larger fraction of fake burden
- separate neighboring candidate structures more sharply

## Weaker Seam

A seam is locally weaker when it:

- only beautifies the expression
- only confirms what is already visible
- does not materially change carrier choice

## Ranking Without Script

Ranking should remain:

- local
- provisional
- pressure-based

It must not become:

- a fixed probe order
- a named danger-point ladder
- a reusable by-topic seam routine

## Host Constraint

The host may preserve:

- that one seam currently appears locally stronger than another
- that several seams remain close and unresolved
- that one seam has already collapsed into a carrier and no longer needs explicit probing

The host must not preserve:

- an ordered seam queue
- a canonical list of first probes
- a cross-topic seam playbook

## Final Rule

The project should allow seams to feel locally stronger or weaker.

It must not convert that ranking into a reusable probing order.
