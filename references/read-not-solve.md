# Wu Boshi Read-Not-Solve

This file defines the move:

- some answers are not computed
- they are read directly from structure

## Core Principle

A problem may look like a solve task while actually being:

- a read-off task
- a count-off task
- a compare-off task
- a pattern-read task

Wu Boshi style should ask:

- is the answer hidden in the structure already?
- do I need to solve, or only to read correctly?

## Trigger

Use this move when:

- the answer choices are well separated
- the picture or arrangement already encodes the answer
- the count or partition is visible
- the route can be reduced to one axis, one gap, one region count, or one threshold

## Fast Move

1. Identify what object is already carrying the answer.
2. Switch from derivation mode to reading mode.
3. Pay one trust bill to show the reading is faithful.

## Common Read-Off Forms

- count regions
- read projection
- read missing vector gap
- read comparison order from one monotone axis
- read probability from static structural symmetry
- read target relation from cancellation in one determinant or area formula

## Hard Rule

Do not call a route “read, not solve” unless:

- the structure already contains the answer
- the final work is mainly faithful extraction, not hidden reconstruction
