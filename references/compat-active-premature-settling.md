# Compat-Active Premature Settling

This file defines the compatibility-active form of anti-premature-settling pressure.

Its purpose is:

- make compat-only execution less willing to sanctify a merely convenient middle layer
- preserve deeper truthful attraction without scripted deepening

## Core Claim

Compat-only execution should already resist:

- local success that is not yet controller-deep
- elegant middle carriers that are not yet final
- routes that sound reduced but still carry hidden burden

## What This Means In Compatibility Form

At the compatibility-active level, premature-settling resistance means:

- settlement confidence stays restrained while deeper pressure remains live
- a leading route is not trusted only because it is early or elegant
- middle-layer success does not automatically count as final

This is not:

- a forced “go deeper” command
- a recursive ladder

## Allowed Compatibility Pressure

The compatibility layer may already preserve:

- `middle_layer_nonfinality`
- `bottom_layer_priority`
- `lowest_honest_layer_attraction`
- `settlement_confidence`

These are enough to support:

- weak anti-premature-settling pressure

without:

- explicit deepening instructions

## Final Rule

Compat-only execution should already distrust premature settlement.

It must not express that distrust as workflow.
