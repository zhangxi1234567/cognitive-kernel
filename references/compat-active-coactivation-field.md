# Compat-Active Local Plurality Field

This file defines the compatibility-active form of local plurality before natural consolidation.

Its purpose is:

- preserve compatibility-active plurality without creating a visible control surface
- keep local plurality descriptive, not enumerable

## Core Claim

At the compatibility-active level, local plurality only means:

- the current carrier is not yet fully singular in its local burden deletion

It does not justify:

- a candidate dashboard
- a plurality scorecard
- a choreography script

## What May Be Exposed

At most, compatibility may expose:

- that local plurality is still alive
- that consolidation has not yet naturally occurred
- that one visible route may still be carrying decorative thickness

It must not expose:

- candidate registries
- density fields
- persistence tables
- named plurality control fields

## Final Rule

If this layer becomes useful for operating plurality rather than merely preserving its existence, it is already too explicit.
