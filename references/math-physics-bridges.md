# Wu Boshi Math-Physics Bridges

This file defines the bridge:

- many mathematical structures become lower-floor and more executable when re-seen through physical meaning

Main slogan:

- derivative -> velocity / acceleration
- monotonicity -> moving forward / backward
- convexity -> acceleration sign
- extremum -> turn-around point
- accumulation -> total traveled amount / accumulated effect

## Core Principle

Math and physics often share the same control skeleton.

The point is not to replace proof with metaphor.
The point is to use the physical meaning to:

- expose the controlling structure early
- kill fake symbolic fear
- choose the right route before formal derivation

## The Running Model

One of the strongest lower-floor bridges is:

- position
- velocity
- acceleration

This lets many derivative problems be renamed as:

- where is the runner?
- is the runner speeding up or slowing down?
- when does the runner stop and turn?
- if two runners start together, who pulls ahead first?

## Common Transfers

### 1. Function Value -> Position

- the function value is where the object currently is

### 2. First Derivative -> Velocity

- sign of `f'` tells direction
- zero of `f'` suggests a stop or direction-change candidate

### 3. Second Derivative -> Acceleration

- sign of `f''` tells whether speed is increasing or decreasing
- convexity / concavity becomes acceleration behavior

### 4. Extremum -> Turn-Around

- maxima and minima become “the moment the direction changes”

### 5. Start Conditions -> Starting-Line Pattern

- `f(0)=0`
- `f'(0)=0`

become:

- same starting position
- same initial speed

Then the decisive issue becomes:

- whose acceleration or higher-order change separates first

## What It Should Not Do

This bridge should not:

- replace legality with vibes
- invent physical meaning where the structure does not match
- skip the formal seal when the route still needs it

## Hard Rule

Use the math-physics bridge when it reduces burden and reveals structure.
Do not keep it if it becomes a decorative analogy instead of a controlling model.
