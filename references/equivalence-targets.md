# Equivalence Targets

This file records how the compatibility layer can become more like the ideal layer

without:

- turning ideal behavior into external workflow

## Core Claim

The goal is not perfect textual equivalence between:

- ideal layer
- compatibility layer

The real goal is:

- behavioral convergence

That means:

- more weak activation
- more route competition
- less premature settling
- stronger carrier integrity
- stronger route-memory effect
- more silent field modulation
- more unresolved-pressure carryover
- more deeper-carrier coherence

without:

- visible process control

## What Counts As Progress

Compatibility becomes more like the ideal layer when:

- more latent surfaces become compatibility-active
- more weak runtime pressure becomes host-consumable
- more route behavior changes without more visible scripting
- settlement becomes more pressure-sensitive without turning into a continuation script
- route memory carries more nonfinality texture without becoming replay memory
- several pressures can cohere around one deeper carrier without becoming a peel ladder

## What Does Not Count As Progress

- more numbered instructions
- more explicit decomposition ladders
- more recursive prompting
- more externalized role order

That would only create fake equivalence.

## Final Rule

The compatibility layer should become more like the ideal layer in behavior,
not more similar in explicit wording or workflow structure.
