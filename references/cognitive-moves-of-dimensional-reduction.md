# Wu Boshi Cognitive Moves Of Dimensional Reduction

This file defines the deepest reusable cognitive moves underneath the visible methods.

Its purpose is:

- not to add more named tricks
- but to explain the mental operations that make many different routes feel playful, transferable, and fast

## Why This Layer Matters

If the library only stores:

- methods
- clusters
- primitives

then it can still feel like a large toolkit.

If the library also stores:

- the small set of cognitive moves behind those tools

then it becomes a thinking engine.

## The Core Cognitive Moves

### 1. Rename The Problem

Do not accept the problem's given identity.

Ask:

- what is this really?

Examples:

- probability story -> matching object
- derivative problem -> motion / boundary / start-line problem
- conic problem -> triangle / distance / ratio problem
- abstract function -> puzzle / archetype problem

Why it works:

- many problems are hard because they are named badly

### 2. Find The Controlling Quantity

Do not track everything.
Find the one quantity that is actually in charge.

Examples:

- one distance
- one difference
- one total
- one ratio
- one parameter
- one seam

Why it works:

- most problems are not equally controlled by all visible parts

### 3. Collapse The Dimension

Turn a bigger object into the smallest truthful object that still decides the answer.

Examples:

- surface -> line
- area -> line-gap / height / projection
- volume -> section / support width
- line -> point
- object -> relation
- process -> static structure
- many variables -> one axis

Why it works:

- humans can play with low-dimensional objects far more easily than high-dimensional clutter

### 4. Match To A Mother Object

Treat novelty as disguise until proven otherwise.

Ask:

- what old object is hiding inside this one?

Examples:

- ellipse -> circle
- strange function -> mother function
- abstract relation -> known behavior family

Why it works:

- memory works by prototypes, not by infinite novelty

### 5. Probe The Weakest Seam

Do not verify everywhere first.
Attack the place that would fail first if the idea were wrong.

Examples:

- boundary
- midpoint
- symmetry point
- special value
- witness case
- degenerating height to 0
- tangent / onset / equality seam

Why it works:

- many global truths are controlled by the first local failure point

### 6. Let The Figure Speak

If the structure can be seen, do not force it to speak only through symbols.

Examples:

- draw the geometry
- draw the motion
- draw the pairing
- read the graph

Why it works:

- visual structure often exposes control faster than symbolic procedure

### 7. Play With A Toy

Shrink the problem to a small, lawful toy before returning to the formal version.

Examples:

- tiny values
- special cases
- degenerate positions
- minimal examples

Why it works:

- toys reveal route shape before formal burden arrives

### 8. Guess Then Pay The Smallest Trust Bill

Let structure guide the guess, then verify only the hinge that matters most.

Examples:

- answer-shape pressure
- route guess
- witness check
- minimal exact seal

Why it works:

- it turns proof search into structured confirmation rather than blind exploration

## Why It Feels Like Play

Problem solving feels like play when:

- the object becomes smaller
- the route becomes visible
- the next move becomes obvious
- the solver is manipulating a control mechanism rather than obeying a heavy procedure

These cognitive moves create that feeling.

Play appears when several moves cooperate:

- rename
- shrink
- match to mother object
- probe a seam
- read instead of rebuild
- seal locally

In the strongest solves, those moves do not appear as isolated tricks.

They often form:

- one continuous dimensional-reduction chain

Use:

- `continuous-dimensional-reduction-chain.md`
- `single-route-elite-execution.md`

to keep the project focused on mature one-route compression rather than route sprawl.

## Mapping Back To The Library

- `Rename The Problem` -> shell-killing families
- `Find The Controlling Quantity` -> target/aggregate/control families
- `Collapse The Dimension` -> normalization / projection / target-only families
- `Collapse The Dimension` also powers area-to-line, volume-to-section, and region-to-boundary readouts
- `Match To A Mother Object` -> prototype / archetype families
- `Probe The Weakest Seam` -> guess / probe / verification families
- `Let The Figure Speak` -> geometry-first and graph-reading families
- `Play With A Toy` -> earliest honest layer / low-floor system
- `Guess Then Pay The Smallest Trust Bill` -> answer-shape / witness / minimal seal families

## Hard Rule

Future additions should not only ask:

- what new trick is this?

They should also ask:

- which cognitive move is this really another surface of?
