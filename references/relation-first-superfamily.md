# Wu Boshi Relation-First Superfamily

This file groups the primitives that all share one deep move:

- read the target through the governing relation before reconstructing the whole object

The point is to unify several routes that can otherwise look separate.

## Core Superfamily Law

Many hard problems are overbuilt because people try to solve the full object.

This superfamily asks first:

- what relation already controls the target?

If that relation is available, spend it before building more machinery.

## Why This Superfamily Matters

Without this file, the library may store these as separate cards:

- `Definition As Direct Readout`
- `Relation Before Object`
- `Target-Only Control`

But the deeper move is one:

- the target is often lower-dimensional than the object that produced it

## Member Primitives

### 1. `Definition As Direct Readout`

Use when:

- the defining law already names the shortest route to the target

### 2. `Relation Before Object`

Use when:

- the target depends on one alignment, ratio, fixed line, sign, or dependence relation, not the full object

### 3. `Target-Only Control`

Use when:

- solving more than the asked quantity would be gratuitous reconstruction

## Shared Trigger Signals

This superfamily is likely live when:

- the target is visibly smaller than the full object
- the statement already contains the law or relation that seems load-bearing
- standard methods would rebuild what the problem has already handed you
- the asked quantity is one line, one ratio, one sign, one dependence claim, or one fixed relation

## Shared Control Mechanism

All members work because:

- the target sits on a lower-dimensional relation
- the relation preserves the admissibility that the target actually needs
- object reconstruction would only restate the same control law later with more cost

## Shared Preserved Structure

Typical preserved items across the superfamily:

- the exact defining or target-relevant relation
- admissibility of the compressed route
- map from relation language to the asked quantity
- legal direct bridge from statement to target

## Shared Failure Modes

This superfamily fails when:

- the relation is necessary but not sufficient
- hidden object details still change the answer
- the compressed relation cannot distinguish branches
- the solver reads the relation too aggressively and skips required legality checks

## Routing Guidance

When the superfamily fires, choose the member primitive like this:

- if the defining law is explicitly present: start with `Definition As Direct Readout`
- if the target is one lower-dimensional connection: start with `Relation Before Object`
- if the danger is over-solving beyond the asked quantity: start with `Target-Only Control`

## Training Guidance

When training this superfamily, do not ask only:

- "did the model solve efficiently?"

Also ask:

- "did the model refuse unnecessary reconstruction?"
- "did it name the relation that actually controls the target?"
- "did it know what still required a minimal seal?"

## Hard Rule

Do not let this superfamily collapse into:

- object worship
- formula accumulation
- orthodox full reconstruction before target control

The law is:

- **relation before object**
