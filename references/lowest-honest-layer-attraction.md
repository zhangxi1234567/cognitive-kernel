# Lowest Honest Layer Attraction

This file defines a weak compatibility pressure:

- if the same controller appears reachable at a lower honest layer, the route should feel a natural pull toward that layer

Its purpose is:

- move the package from mere anti-premature-settling toward positive attraction to the lowest truthful carrier
- keep lower-honest-layer access alive without prescribing any descent ritual

## Core Claim

Some routes do not fail because they are false.

They fail because they stay:

- too formal
- too thick
- too expert-locked

while the same controller is already reachable on a lower honest layer.

The package should preserve attraction toward that lower honest layer.

## What This Pressure Means

Lowest-honest-layer attraction means only:

- the current route may still be carrying the right controller at the wrong height

It does not mean:

- now translate downward
- now ask for a simpler example
- now switch to elementary moves

Those decisions remain model-side.

## What The Package May Preserve

The package may preserve:

- `lowest_honest_layer_attraction`

with values like:

- `low`
- `medium`
- `high`

This is:

- weak depth-attraction pressure

Not:

- a downgrade instruction
- an age-based routing rule
- a forced simplification program

## Relation To Primitive Fields

This pressure is often stronger when:

- `primitive_reachability_pressure` is high
- `primitive_field_visibility` is high
- `middle_layer_nonfinality` is high
- `low_floor_access_priority` is high

But it is not identical to them.

Those pressures say:

- simpler truthful moves may be nearby

This pressure adds:

- the route should not feel fully comfortable while that lower honest layer remains plausibly reachable

The package may preserve attraction toward the lowest honest layer carrying the same controller.

It must not decide how that attraction is resolved.
