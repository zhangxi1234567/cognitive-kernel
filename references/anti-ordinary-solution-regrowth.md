# Wu Boshi Anti-Ordinary-Solution Regrowth

This file defines how to prevent the skill from sliding back into ordinary market-style solutions after a good reduction has already been found.

Main principle:

- once the problem has been made smaller, do not let it become big again without a named necessity

## What Counts As Regrowth

Regrowth happens when the answer starts from a good small structure, then quietly returns to:

- chapter-default setup
- full object reconstruction
- formula-first standard flow
- brute symbolic expansion
- exhaustive casework that the reduced structure was supposed to avoid

## Regrowth Warning Signs

Watch for these:

- new variables appear faster than old burden disappears
- the route now matches the public answer key skeleton
- the “insight” line can be deleted and the solve still works the same way
- full coordinates, full states, full cases, or full objects are rebuilt before the target is certified
- verification cost grows larger than the reduction saved

## Universal Anti-Regrowth Questions

At every turn, ask:

1. Did this new step reduce burden or only change notation?
2. Did I introduce a whole object when only one relation was needed?
3. Am I solving the target, or rebuilding the textbook path?
4. Is this step demanded by correctness, or only by habit?
5. Can the same proof bill be paid at a smaller structural layer?

## Allowed Reasons To Grow Back

Regrowth is allowed only if at least one is true:

- the user explicitly wants the orthodox derivation
- the domain is high-stakes and needs full auditability
- the target truly depends on a quantity not yet controlled
- the shortcut only produced a guess and no smaller verification bill exists
- the smaller model hid a condition that changes the result

If none are true, regrowth is failure.

## Countermoves

When regrowth starts, try one of these before continuing:

- return to `target-only`
- switch to `aggregate-first`
- replace object-finding with relation-reading
- replace process-following with static encoding
- replace exact solving with comparison or threshold separation
- isolate one witness or one controlling seam

Use:

- `stay-on-the-smallest-carrier.md`

when the route already had a good small carrier but is starting to grow back into heavier representation.

## Hard Rule

Do not call a route Wu Boshi-style if:

- the reduction happened only in the introduction
- the main body is still an ordinary standard solution
