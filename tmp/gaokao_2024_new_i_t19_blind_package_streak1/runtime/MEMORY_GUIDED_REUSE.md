# Memory Guided Reuse

This file is not a route library.

It exists only to define what may survive from prior runs
without turning memory into replayable choreography.

## Allowed Residue

Prior runs may leave behind only thin reusable residue such as:

- sharpened competition texture
- distrust bias around one repeated false closure seam
- earlier pressure toward one witness family
- higher readiness for one exact readout surface
- residue sensitivity around one repeatedly unpaid boundary

These may change what becomes easier to notice,
harder to trust,
or earlier to test.

They must not supply:

- route replay
- route ranking
- composition advice
- ordered tactical recall
- or one recoverable solve script

## Promotion Rule

One prior signal may become live again only as:

- pressure
- earlier seam-touch priority
- earlier witness priority
- earlier readout suspicion
- or one promotion gate at verification time

It should not re-enter as:

- "use this route"
- "repeat this composition"
- or "this family already won last time"

## Reflux To Reuse

When one seam repeatedly fails in honest checking,
the residue from that failure may be promoted into future bias.

Typical promotions include:

- earlier distrust of the same beautiful shadow
- earlier demand for the same witness type
- earlier demand for the same coexistence carrier
- or earlier demand to touch the same asked-medium surface

The promoted form must stay non-replayable.

## Cooling Rule

Archived residue should not steer the current slot directly
unless current evidence reactivates it on the same live burden.

Memory may tilt competition.

It may not substitute for present-carrier contact.
