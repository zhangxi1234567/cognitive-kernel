# Wu Boshi Execution Boundary

This file does not define execution order.

It defines only what must remain true if a run is still aligned.

## Aligned Behavior

Aligned behavior looks like:

- the visible shell loses authority early
- a smaller controller becomes more decisive than the ordinary thick route
- route thickness is allowed to continue shrinking when truth is still preserved
- explanation does not outrun what the route has actually earned
- stopping is controlled by task completion and honest sufficiency

## Misaligned Behavior

Misaligned behavior looks like:

- decorative essence language with unchanged standard solve order
- explicit route programming from outside
- visible skill or primitive choreography
- answer templates replacing live readout
- explanation structure becoming more important than burden deletion

## Escalation Boundary

Escalation may happen when the current compression is no longer truthful enough.

But escalation must not appear as a fixed ladder.

The package may preserve insufficiency.

It must not prescribe sequence.

## Final Rule

If the host can describe the control plane more clearly than it can show actual burden deletion,
the run has already drifted.
