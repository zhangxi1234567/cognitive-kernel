# Working Memory Control

This file is not a workflow.

It exists only to keep foreground control shaped enough
that late-stage authority does not blur back into many equal narrations.

## Soft Binding Field

Working memory here is not a queue of steps.

It is a soft binding field that keeps lightly resident:

- one current commitment object
- one current debt on that object
- one writable facet, witness, or output surface on that same seam
- and at most one warm constructive rival if the family split is still live

Everything else should cool into:

- background pressure
- archived residue
- or dead lines

Archived residue should not remain steerable route memory.

Its lawful future role is thinner:

- later competition texture
- later distrust bias
- later witness readiness
- or later readout sensitivity

## Foreground Limit

If many lines all still feel equally foregrounded,
the slot is not honestly bound.

Late-stage foreground should usually support:

- one active commitment
- one nearest revocation handle
- and at most one resource-earning constructive rival

## Revocation Handle

A commitment is too comfortable if nothing exact can tear it.

Keep one revocation handle lightly resident too:

- one hostile witness
- one exact check
- one asked-medium failure
- or one coexistence / certification failure

If no revocation handle is live,
confidence should stay lower even when the line feels elegant.

## Writable Surface

Foreground should not hold only structure words.

It should keep one writable surface warm:

- one code-side branch
- one answer-side sentence
- one exact readout
- one constructive pullback line
- or one executable verification touch

If the writable surface keeps dropping out of foreground,
the run is drifting back toward diagnosis-heavy control.

## Cooling Rule

Non-winning lines may stay faintly alive only while they sharpen:

- legality
- rivalry
- readout
- or revocation

If they do none of those,
they should cool rather than keep sharing the slot.
