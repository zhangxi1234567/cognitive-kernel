# Calibration And Reflux

This file is not a workflow.

It exists only to keep late-stage control honest
about what kind of spend is needed next
and how failure should return locally.

## Separation Versus Materialization

Do not spend as though every remaining burden were the same.

When uncertainty is still live,
prefer spend on:

- the cheapest separating witness
- the cheapest exact check
- the cheapest coexistence carrier
- or the sharpest certification seam

When uncertainty is already low
but the object is still underbound,
prefer spend on:

- field materialization
- split or quotient materialization
- merge or certification materialization
- exact readout materialization
- or asked-medium write

## Resource Gate

A secondary line may keep consuming attention
only if it earns one explicit advantage such as:

- one cheaper falsifier
- one tighter readout
- one smaller coexistence residue
- one sharper certification seam

Without one such advantage,
plurality should cool.

## Asked-Medium Reflux

If one asked-medium touch fails:

- one code branch
- one answer sentence
- one readout
- one constructive line
- one artifact
- or one executable check

do not reopen broad shell analysis by default.

Treat the failure as same-carrier evidence.

Prefer to:

- shrink to the smallest unpaid residue on that same object
- bind one new bite there
- or bind one smaller output program on that same ask

Only regrow the shell if the failure shows
that the carrier itself was misidentified.

## Calibration Rule

Confidence should rise mainly from:

- one survived hostile witness
- one survived exact check
- one reduced residue
- one tighter readout
- or one same-carrier asked-medium cash-back

Do not let fluency, elegance, or family resemblance
outrun calibration.

When the same kind of survived witness, reduced residue, or repeated failure seam
returns across runs,
that pattern may sharpen future salience.

It should re-enter only as thinner competition texture,
not as replayable route advice.
