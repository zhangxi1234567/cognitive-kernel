# Primitive Runtime Coverage

This file is not a solve-time menu.

It exists only to show how much of the archive primitive bank has actually crossed
into the live runtime and action bridge.

Together these are the primitive families that the runtime can currently:

- carry in `primitive_field_if_any`

So the executable core is now **29 canonical primitive families**:

- 11 generic carrier families
- 18 direct skill / primitive-index families

The boundary should stay:

- public kernel first
- direct families second
- archive cards and benchmark lore outside the live solve-time identity

## What "Executable Via Alias" Means

It means the archive card title, alias phrase, and now the literal
`primitive-*.md` filename wrapper can be lawfully recognized by the live runtime
as one of the canonical primitive families.

That recognition may enter through:

- explicit CLI primitive input
- explicit primitive competition candidates
- explicit handoff primitive hints
- or text / pressure inference inside the guard

But the action bridge still lands on the smaller canonical runtime family,
or, when the card already has one, on its own direct executable family.
Hosts should still prefer reading the public kernel interpretation of the live layer first.

## What Is Still Not Implemented

The runtime still does **not** do any of the following:

- infer deep mechanism semantics for every archive primitive from first principles
- automatically regenerate arbitrary new primitive families from solved tasks
- turn the whole research/archive shelf into a solve-time tactic menu
- make every non-primitive `references/*.md` note into an executable route

The live runtime may now carry one thin descriptive semantics layer for its
current executable primitive families.
That does **not** mean the runtime has solved deep mechanism understanding in
the strong sense.
It only means the host can read a smaller mechanism-facing description of what a
live primitive is trying to expose.

So runtime breadth is now much wider, but the whole archive is still not meant to become one flat executable shelf.

## Current Boundary

The project is strongest when:

- the archive stays rich
- the live runtime stays thin
- direct primitive cards cash into action when they are already local
- and generic carrier families still exist for broader non-card execution

The project drifts if:

- every archive note becomes its own route family
- every named document demands its own executor
- or the runtime starts looking like a visible tactic shelf
