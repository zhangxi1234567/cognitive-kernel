# False Essence Elimination

This is not a workflow.

It is a kill layer.

Its purpose is not to generate more candidate essences.

Its purpose is to stop false ones from settling.

Use it quietly when a line feels elegant, deep, or strangely satisfying.

Especially use it when:

- a local pattern looks like the whole law
- a deeper story feels more universal
- a symmetry looks too clean
- a conversion layer looks like the owner of a later metadata failure
- a tiny patch looks right but may still leave the same law open nearby

## Main False-Essence Classes

### 1. False Narrowing

A candidate explains one visible spot but leaves the same witnessed law open on the same boundary.

Typical smell:

- one entry point fixed
- sibling entry points still live
- one visible crack patched
- same invariant still leaking nearby

### 2. False Elevation

A cleaner higher story is chosen even though a lower witnessed boundary already explains and closes the task.

Typical smell:

- architecture story replaces direct witness
- explanation gets prettier
- closure does not improve

### 3. False Deepening

A farther-upstream story feels more foundational, but buys no truer closure for the present witness.

Typical smell:

- more general words
- same actual repair
- no new discriminatory power

### 4. False Symmetry

Symmetry, balance, or family likeness compresses the scene but quietly deletes a real asymmetric edge.

Typical smell:

- elegant cancellation
- beautiful residue/family grouping
- one boundary case suddenly becomes awkward or hand-waved

### 5. False Completion

A mechanism story sounds finished, but it has not cashed back into the exact asked interface.

Typical smell:

- diagnosis without exact patch
- controller without explicit answer
- deep claim without concrete reconstruction

## Kill Questions

Do not ask all of these mechanically.

Ask the one most likely to kill the current impostor.

- If this candidate were wrong, what is the cheapest witness that would break it first?
- What nearest rival explanation is still alive, and what single seam separates them?
- Did this proposed essence change the live variables, operator choice, repair surface, or closure path, or did it only sound deeper?
- If the story is symmetric, what asymmetric edge, endpoint, branch, or ownership case might survive the symmetry?
- If this is a metadata bug, is this layer merely carrying the metadata, or does it own the exact legality test for keeping it?
- If both a broad validator and a narrower final pinning owner exist, which one still has the last narrow discretion over the finished result?
- If this is a boundary bug, does the same law remain live across sibling entry points?
- If this is a deeper story, what present-task closure does it buy that the nearer boundary did not?
- If this candidate wins, can it cash back into the exact asked interface right now?

## Settlement Rule

Do not let a candidate settle merely because it is:

- elegant
- deeper
- more universal
- more mathematical
- more architectural
- more compressed

Let it settle only after it survives one honest kill attempt against its nearest live rival.

If no rival was ever identified, confidence should stay lower.

If no falsifiable seam was ever touched, essence has not been earned.

## Conflict Priority

When good-looking candidates conflict:

1. witnessed falsifier beats elegance
2. exact asked-interface closure beats beautiful nearby explanation
3. lowest complete boundary beats higher clean story
4. legality-test owner beats retention layer
5. validator beats producer
6. one complete invariant repair beats one-site patch
7. narrower final pinning owner beats broader validator when both can still reject the carried metadata

## Final Warning

If this layer becomes a ritual, it has failed.

If it silently kills impostors and leaves one controller standing,
it is doing its job.
