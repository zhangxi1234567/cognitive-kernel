# Research Layer

This directory boundary exists even though much legacy material still lives in `references/`.

## Purpose

Research material is allowed to be:

- explicit
- comparative
- historical
- archive-heavy
- name-rich
- seed-driven

because research asks:

- where did a move come from?
- how often did it recur?
- is it transferable?
- what are the failure modes?

## Runtime Warning

Research language is not runtime language.

Useful research artifacts often become harmful when exposed directly at solve time, especially:

- primitive registries
- topic-routing maps
- candidate ranking systems
- tactic trees
- solve protocols
- training bundles

Those objects help analysis.

They also risk turning a live metacognitive layer back into an external menu.

## Current Status

For now, the large legacy archive remains in:

- `references/`

Treat that directory as research/archive by policy, even before a physical migration is complete.

## Migration Rule

When deciding whether a document should move closer to runtime, ask:

1. Does it change attention rather than prescribe steps?
2. Would it still make sense outside school-math phrasing?
3. Can it survive without named tactic recall?
4. Would exposing it make the run more native, or just more scripted?

If the answer to the last question is "more scripted," keep it here.
